## הוראות התקנה והרצת הבדיקות

על מנת להריץ את כל הבדיקות שנכתבו בפרויקט זה, יש לעקוב אחר השלבים הבאים:

1. **שכפול הפרויקט:**
   פתחו את הטרמינל והריצו את הפקודה הבאה כדי לשכפל את הפרויקט לסביבת העבודה שלכם:
   `git clone <your_repository_address>`

2. **פתיחת הפרויקט:**
   פתחו את הפרויקט המשוכפל בסביבת הפיתוח (IDE) המועדפת עליכם לפיתוח ב-Java (לדוגמה IntelliJ IDEA או Eclipse).

3. **הרצת הבדיקות:**
   הפרויקט מבוסס על סביבת עבודה עם Gradle. ניתן להריץ את כלל בדיקות היחידה (שנכתבו תחת קובץ ה-AppTest) דרך ממשק ה-IDE, או לחלופין דרך הטרמינל על ידי הרצת הפקודה הבאה:
   `./gradlew test`

## פירוט הפונקציות וכיסוי הבדיקות

### הסט של ארין (סט 1) - פונקציות מתמטיקה ושגיאות

* **חיבור שני מספרים**
  * **שם הפונקציה:** `add(int a, int b)`
  * **מקרה נורמטיבי:** חיבור שני מספרים חיוביים.
  * **מקרי קצה:** חיבור מספר חיובי ושלילי, חיבור של שני אפסים, חיבור של מספר ואפס.

* **בדיקת מספר ראשוני**
  * **שם הפונקציה:** `isPrime(int n)`
  * **מקרה נורמטיבי:** זיהוי מספר ראשוני (אמת) וזיהוי מספר פריק (שקר).
  * **מקרי קצה:** מספרים קטנים מ-2 (1, 0, ומספרים שליליים) כדי לוודא שמוחזר `false`.

* **חישוב עצרת**
  * **שם הפונקציה:** `factorial(int n)`
  * **מקרה נורמטיבי:** עצרת של מספר רגיל וחיובי.
  * **מקרה קצה:** עצרת של 0 (מוודא שמוחזר 1).
  * **מקרה שגיאה:** קלט שלילי. נעשה שימוש ב-`assertThrows(IllegalArgumentException.class)` כדי לוודא זריקת שגיאה.

* **חישוב ממוצע של מערך**
  * **שם הפונקציה:** `average(int[] arr)`
  * **מקרה נורמטיבי:** מערך עם מספרים שלמים רגילים.
  * **מקרי קצה:** מערך שבו הממוצע אינו שלם (תוצאה עם נקודה עשרונית).
  * **מקרה שגיאה:** מערך ריק. נעשה שימוש ב-`assertThrows(IllegalArgumentException.class)` כדי לוודא זריקת שגיאה.

### הסט של מיכל (סט 2) - פונקציות מחרוזות ורשימות

* **היפוך מחרוזת**
  * **שם הפונקציה:** `reverse(String s)`
  * **מקרה נורמטיבי:** מחרוזת טקסט רגילה.
  * **מקרי קצה:** מחרוזת ריקה (""), מחרוזת עם תו בודד.

* **בדיקת פלינדרום**
  * **שם הפונקציה:** `isPalindrome(String s)`
  * **מקרה נורמטיבי:** מחרוזת שהיא פלינדרום פשוט (אמת) ומחרוזת שאינה פלינדרום (שקר).
  * **מקרי קצה:** פלינדרום עם רווחים ואותיות גדולות (הפונקציה מנקה אותם ולכן מוודאים שהזיהוי ממשיך לעבוד כראוי).

* **בדיקת אנגרמה**
  * **שם הפונקציה:** `isAnagram(String s1, String s2)`
  * **מקרה נורמטיבי:** שתי מחרוזות שהן אנגרמה (אמת) וכאלו שאינן (שקר).
  * **מקרי קצה:** אנגרמה עם אותיות גדולות/קטנות ורווחים (מוודא התעלמות ממקרי Case-sensitivity ורווחים).

* **סינון מספרים זוגיים מרשימה**
  * **שם הפונקציה:** `filterEvens(List<Integer> list)`
  * **מקרה נורמטיבי:** רשימה מעורבת של מספרים זוגיים ואי-זוגיים.
  * **מקרי קצה:** רשימה של מספרים זוגיים בלבד, רשימה של מספרים אי-זוגיים בלבד (החזרת רשימה ריקה), ורשימה ריקה לגמרי.

---

### הסט של מריאנה (סט 3) - פונקציות מבני נתונים ומפות

* **הפקת רשימת מספרי פיבונאצ'י עד גבול מסוים**
  * **שם הפונקציה:** `fibonacciUpTo(int n)`
  * **מקרה נורמטיבי:** גבול רגיל ליצירת סדרה (לדוגמה 10, ולוודא חזרת הרשימה המדויקת).
  * **מקרה קצה:** גבול של 0 (מוודא חזרת רשימה עם `[0]`).
  * **מקרה שגיאה:** קלט שלילי. נעשה שימוש ב-`assertThrows(IllegalArgumentException.class)` כדי לוודא זריקת שגיאה.

* **ספירת תדירות של תווים במחרוזת**
  * **שם הפונקציה:** `charFrequency(String input)`
  * **מקרה נורמטיבי:** מחרוזת עם מספר אותיות חוזרות ומיפוי מדויק של הערכים והמפתחות ב-Map.
  * **מקרי קצה:** מחרוזת ריקה (מוודא חזרת מפה ריקה).

* **מציאת המילה הנפוצה ביותר בטקסט**
  * **שם הפונקציה:** `mostCommonWord(String text)`
  * **מקרה נורמטיבי:** טקסט תקין שבו מילה אחת מופיעה בבירור הכי הרבה פעמים.
  * **מקרי קצה:** טקסט עם סימני פיסוק ואותיות גדולות (מוודא התעלמות וזיהוי מדויק של המילה הנפוצה).
