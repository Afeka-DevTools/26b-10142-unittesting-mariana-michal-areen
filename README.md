# [cite_start]פרויקט בדיקות יחידה - תרגיל בית 2: כלי פיתוח [cite: 2]

[cite_start]פרויקט זה מכיל בדיקות יחידה לפונקציות השונות המצויות בקובץ `App.java`[cite: 34]. [cite_start]הבדיקות נכתבו תוך שימוש בפונקציות ה-Assert השונות בקובץ ה-`AppTest`[cite: 35]. 

## חברות הצוות וחלוקת העבודה
[cite_start]העבודה חולקה בין חברות הצוות, כאשר כל אחת כיסתה מקרי בסיס, מקרי קצה ושגיאות כדי לוודא שכל הנתיבים נבדקו[cite: 49]:

* **ארין (סטודנטית א') - פונקציות מתמטיקה ושגיאות:**
  * `add(int a, int b)`: נבדק חיבור של מספרים חיוביים (מקרה נורמטיבי), חיובי ושלילי, ושילובים עם אפס (מקרי קצה).
  * `isPrime(int n)`: נבדקו מספרים ראשוניים ופריקים (נורמטיבי), ומספרים קטנים מ-2 כולל מספרים שליליים שמוודאים החזרת false (קצה).
  * `factorial(int n)`: נבדקה עצרת למספר חיובי (נורמטיבי) ול-0 (קצה), ווידוא זריקת שגיאת `IllegalArgumentException` עבור מספר שלילי.
  * `average(int[] arr)`: נבדק ממוצע של מערך רגיל, מערך שתוצאתו שבר (קצה), ווידוא זריקת `IllegalArgumentException` עבור מערך ריק.

* **מיכל (סטודנטית ב') - פונקציות מחרוזות ורשימות:**
  * `reverse(String s)`: נבדק היפוך מחרוזת רגילה (נורמטיבי), מחרוזת ריקה ומחרוזת עם תו בודד (קצה).
  * `isPalindrome(String s)`: נבדקו פלינדרומים תקינים ושגויים, כולל מקרים של רווחים ואותיות גדולות שהפונקציה מנקה לפני הבדיקה.
  * `isAnagram(String s1, String s2)`: נבדק זיהוי אנגרמות תקינות ושגויות, כולל התעלמות מרווחים ואותיות גדולות.
  * `filterEvens(List<Integer> list)`: נבדק סינון רשימות מעורבות, רשימות המכילות רק זוגיים או רק אי-זוגיים, ורשימות ריקות.

* **מריאנה (סטודנטית ג') - פונקציות מבני נתונים ומפות:**
  * `fibonacciUpTo(int n)`: נבדקה יצירת רשימה עד גבול רגיל ועד 0 (קצה), ווידוא זריקת שגיאת `IllegalArgumentException` לקלט שלילי.
  * `charFrequency(String input)`: נבדק המיפוי של תדירות אותיות במחרוזת רגילה, ובמחרוזת ריקה שאמורה להחזיר מפה ריקה.
  * `mostCommonWord(String text)`: נבדק זיהוי המילה הנפוצה ביותר תוך התעלמות מסימני פיסוק ואותיות גדולות (קצה).

## הוראות הרצה למשתמש
[cite_start]כדי להריץ את כל הבדיקות שנכתבו בפרויקט זה משלב השכפול ועד להרצה בפועל, יש לעקוב אחר השלבים הבאים[cite: 39]:

1. [cite_start]**שכפול הפרויקט:** פתחו את הטרמינל והריצו את הפקודה הבאה כדי לשכפל את הפרויקט לסביבת העבודה שלכם[cite: 21]:
   [cite_start]`git clone <your repository address>` [cite: 22]
2. [cite_start]**פתיחת הפרויקט:** פתחו את הפרויקט המשוכפל בסביבת הפיתוח (IDE) המועדפת עליכם לפיתוח ב-Java[cite: 29].
3. [cite_start]**הרצת הבדיקות:** הפרויקט מבוסס על סביבת עבודה עם Gradle[cite: 15]. [cite_start]ניתן להריץ את כלל בדיקות היחידה דרך ממשק ה-IDE (הרצת קובץ ה-`AppTest` [cite: 35]), או לחלופין דרך הטרמינל על ידי הרצת פקודת הבדיקות של Gradle (לרוב `gradlew test` או `gradle test`).
